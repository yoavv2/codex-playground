## tasks

### Task 1: Establish route map and app layout

1. Implement Expo Router layout files and route groups for the Phase 03 screen set.
2. Define route conventions that match upcoming Phase 04/05 needs:
   - Connection
   - Threads
   - Thread Detail (`threadId` route param)
   - Settings
3. Ensure navigation structure is clear and minimal (no feature creep).

Deliverable:
- Stable route topology exists and is ready for screen skeleton content.

### Task 2: Implement screen skeleton components

1. Create each Phase 03 screen with:
   - title/header text
   - placeholder body blocks that indicate future integration points
2. Add a basic thread item -> thread detail navigation path.
3. Keep screen state local/minimal and avoid API coupling.

Deliverable:
- Users can navigate among all required Phase 03 screens.

### Task 3: Verify navigation behavior and handoff to 03-03

1. Run static checks for the mobile package.
2. Start app in non-interactive mode to ensure route compile/startup succeeds.
3. Record execution notes in `03-02-SUMMARY.md`, including what `03-03` should wire next.
4. Update `.planning/STATE.md` next action to execute `03-03-PLAN.md`.

Deliverable:
- Navigation shell verified and ready for persistence/connection behavior in `03-03`.
