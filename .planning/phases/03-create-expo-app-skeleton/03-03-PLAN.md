## tasks

### Task 1: Implement typed settings model and persistence layer

1. Create a typed connection settings model with:
   - server URL
   - auth token
   - optional profile label
2. Implement storage with secure handling for token values.
3. Add load/save methods suitable for app startup hydration.

Deliverable:
- Reusable settings module with clear typing and secure token persistence.

### Task 2: Wire Connection screen to persistence lifecycle

1. Hydrate Connection screen from persisted settings on load.
2. Wire form edits and save action to persistence layer.
3. Provide clear local UI feedback for save/hydration state.

Deliverable:
- Settings entered in Connection screen persist across app restarts.

### Task 3: Add `/api/health` test action and status feedback

1. Implement a lightweight health-check action from Connection screen.
2. Report success/failure status in a way that is useful for setup debugging.
3. Keep networking implementation intentionally simple and Phase 04-compatible.

Deliverable:
- Connection screen can validate server reachability with user-visible result.

### Task 4: Finalize docs and planning handoff

1. Update `farfield/README.md` with a concise mobile startup section:
   - workspace commands
   - expected secure server setup assumptions
2. Create `.planning/phases/03-create-expo-app-skeleton/03-03-SUMMARY.md`.
3. Update `.planning/STATE.md` and `.planning/ROADMAP.md`:
   - Phase 03 plan/summary counts
   - next action for Phase 04

Deliverable:
- Phase 03 is complete and Phase 04 can begin immediately.
